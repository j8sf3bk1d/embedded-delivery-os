# Source Authority Template

## Primary source of truth

List the exact files that define:

- the discovery brief if no formal PRD exists yet
- product intent
- scope and non-goals
- critical constraints
- canonical process rules

## Implementation-boundary docs

List the derived docs future contributors should read before raw source materials:

- working distillation
- landmine or risk map
- architecture notes
- test matrix
- experience model when UX-heavy

## Background or secondary docs

List useful context that should not silently override the primary source of truth.

## Archive or provenance-only docs

List raw imports or historical docs kept for traceability only.

## Cold-start reading order

Document the exact order a zero-context contributor should follow.

If no formal PRD exists yet, call out the discovery brief explicitly.
