# Decisions

Use this folder for concise ADR-style records when future contributors would reasonably ask, "Why was it done this way?"

Recommended filename pattern:

- `YYYY-MM-DD-short-title.md`

Good decision-record candidates:

- architecture or runtime posture
- data ownership boundaries
- auth or trust-boundary choices
- dependency or package-manager posture
- external-service decisions
- meaningful UX or workflow commitments with technical impact

Use [docs/process/templates/decision-template.md](../../process/templates/decision-template.md) as the default starting point.
