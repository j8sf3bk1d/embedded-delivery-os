# Issues

Use this file for known problems, unresolved risks, deferred bugs, review findings that survive the current phase, and gaps that need follow-up.

Use this as a lightweight risk register, not a dumping ground.

Suggested format:

| Status | Severity | Area | Issue | Owner | Related docs or files | Next checkpoint | Notes |
|---|---|---|---|---|---|---|---|
| open | medium | example | Replace with a real issue | unassigned | `docs/...` or `src/...` | next phase closeout | mitigation, rationale, or next step |

Status guidance:

- `open`
- `in progress`
- `deferred`
- `resolved`
- `accepted`

Routing guidance:

- use the active phase doc for immediate same-phase follow-up
- use this file for durable deferred work
- create a decision record when a risk forces a durable architecture, security, or operational choice
- create a new phase when the fix is large enough to deserve its own bounded work item
