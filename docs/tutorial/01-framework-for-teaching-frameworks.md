# Framework For Teaching Frameworks

This tutorial track uses one repeatable teaching structure for each framework or sub-framework.

The point is to make process memorable by tying it to the pain that created it.

This is the intentionally meta lesson in the tutorial.

One influence on this teaching shape is Russell Brunson's framing around the "epiphany bridge": make the painful discovery legible before presenting the mechanism. This repo adapts that idea for teaching process frameworks rather than marketing offers.

## Teaching Shape

For each framework, teach it in this order:

1. how it was learned or earned
2. the strategy
3. the tactics
4. the proof or belief builders

## 1. How It Was Learned Or Earned

This is the pain of discovery.

Document:

- the recurring failure mode
- what kept going wrong
- why the usual "just be more disciplined" answer was not enough
- what finally made the missing mechanic obvious

This is the epiphany bridge. Without it, the process reads like arbitrary preference.

## 2. The Strategy

This is the governing idea.

Document:

- what the framework is trying to prevent
- what it optimizes for
- what tradeoff it accepts
- what it refuses to do

This keeps the framework from collapsing into disconnected tactics.

## 3. The Tactics

This is the concrete implementation layer.

Document:

- the actual repo artifacts
- the triggers that make the framework apply
- the checklists or reviews that keep it alive
- the boundaries between this framework and adjacent ones

This is where the framework stops being theory and starts affecting behavior.

## 4. The Proof Or Belief Builders

This is how the reader updates their belief that the framework matters.

Document:

- what failure mode it catches
- what quality improves when used correctly
- what gets easier for a cold-start contributor or agent
- what real repo artifact proves it is embedded instead of aspirational

## Why This Structure Matters

Most process explanation starts at tactics:

- "here is the checklist"
- "here is the doc template"
- "here is the review gate"

That is the least persuasive place to start.

Starting with the earned pain creates motivation. Strategy creates coherence. Tactics create actionability. Proof creates trust.

## Tutorial Authoring Rule

When writing a lesson in this repo:

- do not present the tactic before the pain
- do not present the process as universal law
- do not hide the tradeoffs
- do point to the concrete artifact that embodies the lesson
