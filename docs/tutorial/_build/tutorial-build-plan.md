# Tutorial Build Plan

Purpose:
Plan how the final tutorial set gets compiled from the framework inventory, story bank, and lesson requirements.

## Compilation model

### Inputs

- the repo-native operating artifacts
- the audience model
- the failure-family map
- the story bank

### Intermediate structures

- framework inventory
- lesson candidates
- per-lesson story + tactic alignment
- audience read paths

### Outputs

- final lesson pages near the root of `docs/tutorial/`
- supporting maps and planning docs in `docs/tutorial/_build/`

## Suggested lesson sequence

### Umbrella lesson

1. Why Projects Need An Embedded Workflow Operating System

### First wave

1. Why Reading Paths Beat Repo Tours
2. Why Source Authority Must Exist Before Implementation
3. Why Active Work Cannot Live Only In Chat Memory
4. Why UX Keeps Regressing Into Screen Soup
5. Why Reviews Need Routing, Not Just Findings
6. Why "Works Locally" Is Not A Release Process

### Second wave

- Why Closeout Hygiene Matters
- Why Progressive Disclosure Beats Flat Operational Surfaces
- Why External Resources Need A Human-Executable Checklist
- Why Decision Records Should Be Triggered, Not Overused

Status:

- complete

## Build rules

- final lessons should feel public-facing, not like raw planning notes
- `_build/` docs should remain explicit planning material
- not every framework in the prompt needs to become its own lesson
- combine sub-frameworks when the pain and strategy are the same
