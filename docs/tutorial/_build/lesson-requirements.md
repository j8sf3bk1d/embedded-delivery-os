# Lesson Requirements

Purpose:
Define what a final lesson needs before it is considered ready to publish in the tutorial root.

## Minimum requirements for every lesson

- a named framework or sub-framework
- a recognizable failure mode
- a clear strategy, not just tactics
- concrete repo artifacts that embody the tactic layer
- proof or belief builders showing what behavior changes

## Required sections

1. how it was learned or earned
2. the strategy
3. the tactics
4. the proof or belief builders

## Audience requirements

Each lesson should be understandable to:

- managers
- developers
- non-developers

This does not require three separate versions, but it does require:

- minimal jargon at the start
- one or more concrete examples
- visible translation from pain to process mechanic

## Repo-artifact requirements

Each lesson should point to:

- the canonical operating docs or templates involved
- the review or lifecycle trigger if relevant
- the specific artifact that proves the framework is embedded

## Exclusion rule

Do not publish a final lesson if it is only:

- abstract theory
- generic process advice
- a bag of tactics with no unifying strategy
- a story with no repo-native artifact behind it
