# Tutorial

This folder is the public-facing teaching track for Embedded Delivery OS.

It is not the canonical operating surface for day-to-day project work. It exists to explain why the operating system in this repo exists, what problems it solves, and how its mechanics fit together.

If you are here to adopt the scaffold for a real project, start with [../process/first-real-use.md](../process/first-real-use.md) instead of this folder.

## Teaching Model

Each lesson should teach one named framework or sub-framework using the same shape:

1. how it was learned or earned
2. the strategy
3. the tactics
4. the proof or belief builders

The goal is not to present process as abstract best practice. The goal is to show which failure mode or recurring pain forced the workflow to exist.

## Build Layer

The tutorial is developed progressively.

The source structure for that work lives in:

- [_build/README.md](_build/README.md)

That folder contains the audience model, framework inventory, failure-family map, story bank, lesson requirements, and tutorial build plan used to compile the final public lessons.

## Authoring Rules

- keep lessons concrete
- start from the pain, not the mechanism
- prefer one framework per lesson
- link to the repo-native artifact that embodies the lesson
- use examples from real project drift, not hypothetical generic teams
- keep operating docs and tutorial docs separate; tutorial docs explain the system, operating docs run the system
- keep final lessons near the tutorial root; keep planning and source structures in `_build/`
